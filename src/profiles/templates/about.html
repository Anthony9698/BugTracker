About Bug Tracker

Bug Tracker is an application to track errors in software. It was created by Anthony Viera over the summer of 2020.

Through the use of a ticketing system, Bug Tracker is able to record bugs or issues in a software development project. It also ensures user and role level security to ensure only authorized users can access tickets and projects.

This application was built using Django MVC and Python with a Postgres server database. Development was completed with Visual Studio Code 2019. Bug Tracker's webpages use HTML5 and Bootstrap 4.

Roles: Bug Tracker has four roles that users can be assigned to. Users can have multiple roles and are automatically assigned as a Submitter upon registration.

Below are brief descriptions of each role's permissions.

Submitter
  • Create new tickets for assigned projects
  • View and edit tickets they have submitted

Developer
  • View and edit tickets they have submitted

Project Manager
  • View and edit tickets for projects they are assigned to
  • Create new projects
  • Assign users to projects
  • Assign a developer to a ticket
  • Archive projects

Admin
  • Assign users to roles
  • View and edit all tickets
  • Create new projects
  • Assign users to projects
  • Archive projects

For each role, users can add comments or attachments to tickets that have access to.

Ticket priorities: A ticket can have a Low, Medium, High, or Critical priority.

Ticket statuses: A ticket can have any of the five following statuses:

  • New: Automaically assigned this status on creation
  • Waiting for support: Automatically assigned this status when assigned to a developer
  • Waiting for customer: Ticket is waiting on a response from a third-party source
  • On hold: Work on the ticket has been suspended
  • Resolved: The issue raised in the ticket has been resolved

Ticket types: A ticket can be of type Error report, Service request, Feature request or Other.

FAQs

How do I submit a new ticket?

Make sure you are assigned the Submitter role and have been assigned to the project. If not, contact your Admin. Navigate to your Tickets page and click on the Create New link to fill out the ticket details and submit the ticket.

When are email notifications sent out?

Email notifiations are sent to developers on the following events:

  • Developer assigned to ticket
  • Ticket reassigned to another developer
  • User other than the developer edits ticket
  • User other than developer adds a comment or attachment

Can I change my user name and password?

Yes, any user can edit his or her first and last name, email and/or password. Go to Settings on the top menu to change these.

How do I get my role changed?

Contact you Admin to change your role.
